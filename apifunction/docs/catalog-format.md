# GovOpenAPI Catalog Format

`data/master_catalog.jsonl` is the master table for supported government Open APIs.
Each row is intended to represent one source dataset or statistic, not just one source endpoint.

Each record includes:

- `index`: stable unique id such as `source:table_name`
- `dataset_id`: source-native dataset/statistic id
- `parent_id`: optional parent node id for source trees
- `service_id`: optional service code used by the real fetch API
- `table_name`: human-friendly dataset or endpoint name
- `source`: source system name like `ecos`, `kosis`, or `molit`
- `description`: short searchable summary
- `endpoint`: request URL or URL template
- `method`: HTTP method
- `auth_type`: `none`, `query_key`, `header_key`, or `path_key`
- `metadata`: source-specific fields that do not fit the normalized core schema
- `auth_env_var`: preferred environment variable for the API key
- `auth_query_name` / `auth_header_name`: where the key should be injected
- `tags`: free-form search tags
- `params`: structured parameter schema

Parameter items support:

- `name`
- `required`
- `type`
- `default`
- `enum`
- `description`
- `source_alias`

Recommended credential resolution order:

1. explicit function argument
2. environment variable such as `GOVAPI_KOSIS_KEY`
3. `~/.govapi/credentials.toml`

Example `credentials.toml`:

```toml
GOVAPI_KOSIS_KEY = "..."
GOVAPI_ECOS_KEY = "..."
```

Source-level raw catalogs live under `data/source_catalogs/*.jsonl`.
`govopenapi/update/source_collectors.py` can refresh source catalogs from live endpoints and
`govopenapi/update/build_catalog.py` merges them into the master catalog.

Current practical approach:

- manageable sources such as `ECOS` can be checked into source catalogs directly
- large sources such as `KOSIS` should be generated on demand from the collector
- sources whose public listing endpoints are incomplete may need manual seed files or a custom scraper
- the current generated catalog includes `ECOS`, `OpenFiscal`, `MOLIT`, `IMF`, `eNara`, and `World Bank`
- `MOLIT` rows are generated by walking the public category AJAX tree and enriching each dataset with `statView.do` metadata such as `form_id`, `style_num`, and supported date values
